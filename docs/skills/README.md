## What Are Skills?

**Skills** are focused, self-contained guides for specific topics in frontend development. Each skill:

- ✅ Covers ONE topic in depth
- ✅ Includes real, practical examples
- ✅ Takes 1-4 hours to master
- ✅ Links to related skills
- ✅ References official patterns

Instead of one huge 5,000-line documentation file, we have 12 focused skills + supporting guides.

## Learning Paths

### Path 1: New Developer (5 hours)
```
Week 1 (Day 1): Complete Tier 1
  ├─ 03-DEVELOPMENT-WORKFLOW (30 min)
  ├─ 01-COMPONENT-ARCHITECTURE (2-3 hours)
  └─ 02-NAMING-CONVENTIONS (1-2 hours)

Week 2 (Days 3-5): Start Tier 2
  ├─ 04-TESTING-PATTERNS (3-4 hours)
  ├─ 05-STATE-MANAGEMENT (2-3 hours)
  └─ 06-STYLING-CONVENTIONS (2 hours)
```

### Path 2: Feature Sprint (3 hours)
```
Before starting feature:
  ├─ 01-COMPONENT-ARCHITECTURE (skim - 15 min)
  ├─ 05-STATE-MANAGEMENT (review - 30 min)
  ├─ 07-API-INTEGRATION (if backend calls)
  ├─ 04-TESTING-PATTERNS (write tests - varies)
  └─ 09-ERROR-HANDLING (error cases - 1 hour)
```

### Path 3: Code Review (1 hour)
```
When reviewing PR:
  ├─ 12-CODE-REVIEW-CHECKLIST (2-5 min per section)
  ├─ Relevant skills as needed
  └─ Reference 02-NAMING-CONVENTIONS for naming issues
```

### Path 4: Performance Optimization (2 hours)
```
When app is slow:
  ├─ 10-PERFORMANCE-BEST-PRACTICES (detailed read)
  ├─ 04-TESTING-PATTERNS (test performance)
  └─ 06-STYLING-CONVENTIONS (CSS optimization)
```

## FAQ

**Q: Should I read all 12 skills?**
> No! Read what you need for your current task. For new developers, start with Tier 1.

**Q: Can I skip a skill?**
> Yes, if you already know the topic. But skim it to check for project-specific patterns.

**Q: Are these skills like courses?**
> Not really! They're reference guides you use while coding, not linear courses to complete.

**Q: How do I remember all these?**
> You don't! Bookmark this folder and reference skills as needed. You'll naturally memorize patterns over time.

**Q: Can I use other resources?**
> Yes! These skills are project-specific. External resources (React docs, MDN, Bootstrap docs) are always helpful.
